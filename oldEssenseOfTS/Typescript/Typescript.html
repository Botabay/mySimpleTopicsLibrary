✅Typescript (теория)

TypeScript — язык программирования, представленный Microsoft в 2012 году и позиционируемый как средство разработки веб-приложений, расширяющее возможности JavaScript

TypeScript отличается от JavaScript возможностью явного статического назначения типов, поддержкой использования полноценных классов (как в традиционных объектно-ориентированных языках), а также поддержкой подключения модулей, что призвано повысить скорость разработки, облегчить читаемость, рефакторинг и повторное использование кода, помочь осуществлять поиск ошибок на этапе разработки и компиляции, и, возможно, ускорить выполнение программ.

🔰 3) Типизация state

rootReducer - это функция, которая возвращает state всего нашего приложения. Именно то, что нам и нужно.

И чтобы вернуть то, что возвращает функция воспользуемся ReturnType

Чтобы динамически создать тип который нам возвращает rootReducer, воспользуемся оператором ReturnType

Оператор ReturnType анализирует тип переданный в нее функции (rootReducer) и берет ее возвращаемый тип

🔰 4) Типизация классовых компонент

type StateType = {

   // описываем локальный стейт

   id: string

}

type OwnPropsType = {

   // описываем пропсы, которые пришли из родительской компоненты

   id: string

}

type MapStatePropsType = {

   // описываем, что возвращает MapStateToProps

   todolists: ITodo[]

}

type MapDispatchPropsType = {

   // описываем, что возвращает MapDispatchToProps

   getTodo: () => void;

   addTodo: (todolistTitle: string) => void;

}

type PropsType = OwnPropsType & MapDispatchPropsType & MapStatePropsType

class App extends React.Component<PropsType, StateType> {

   state: StateType = {

       error: false,

       title: ""

   };

   return ( <Todolist/> );

}

const mapStateToProps = (state: AppStateType): MapStatePropsType => {

   return {

        todolists: state.todolistReducer.todolists

   }

};

// connect

export default

connect<MapStatePropsType, MapDispatchPropsType, OwnPropsType, AppStateType>(

       mapStateToProps, {getTodlists, addTodolist, setTodolist})

)(App)

!Важно

При описании типизации функции connect в компоненте может не быть OwnPropsType или MapDispatchPropsType или MapStatePropsType. В этом случае на их месте указываем пустой объект {} соблюдая последовательность передаваемых параметров.

Пример:

connect<{}, MapDispatchPropsType, {}, AppStateType>(

       mapStateToProps, {getTodlists, addTodolist, setTodolist})

)(App)

Статья на которую ссылается Димыч в видосе  https://habr.com/ru/company/alfa/blog/452620/

1) Собираем все action creator в объект actions

2) Соответственно во всех местах, где мы диспатчим экшены, мы теперь обращаемся к определенному свойству объекта actions

3) Теперь нужно написать вспомогательную функцию, которая будет описывать типизацию объекта actions автоматически

Переходим в файл где мы создаем store т.к. нам этот тип будет нужен для всех редьюсеров и создаем InferActionTypes

// infer - додумывать, подразумевать


type InferActionTypes<T> = T extends { [keys: string]: (...args: any[]) => infer U } ? U : never*


// Если тип переданный в T является объектом у которого ключ равен строке, а значением является функция, тогда определи (додумай) возвращаемый тип U с помощью оператора infer (где U - возвращаемый action) и верни его, в противном же случае не возвращай ничего (never)

 

*Разница между void и never

https://ru.stackoverflow.com/questions/684964/%D0%92-%D1%87%D0%B5%D0%BC-%D1%80%D0%B0%D0%B7%D0%BD%D0%B8%D1%86%D0%B0-%D0%BC%D0%B5%D0%B6%D0%B4%D1%83-void-%D0%B8-never-%D0%B2-typescript

 

4) Теперь возвращаемся в редьюсер и описываем типы для всех наших экшенов вот так:


type ActionTypes = InferActionTypes<typeof actions>

5) И чтобы сейчас в кейсах редьюсер понимал, что в него приходит, нам нужно сделать возвращаемый объект (он же action) как константу (as const)

6) Т.к. теперь у нас есть автоматическая типизация, то если мы в type запишем строку, удалив при этом константы, мы не сможем допустить ошибки

Поэтому Удаляем константы и приписываем в action type строковые значения

7) И после этого в кейсах я не могу ошибиться, т.к. TypeScript показывает что можно прописать
